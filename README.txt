PFormat

・概要
  ・フィールドとフォーマットを組み合わせて文字列を作成します。
・動作環境
  ・Microsoft Windows 10
  ・.NET Framework 4.8
・ファイル
  ・PFormat.exe
      ・実行ファイルです。
  ・README.txt
      ・説明書 (このファイル)
・使い方
  ・PFormat では、設定した可変値"フィールド"と、フィールドを埋め込む文章"フォーマット"の組み合わせを"グループ"として管理します。
  ・左上のコンボボックスは、グループを選択・管理します。
    ドロップダウン リストからグループを選択すると、グループを切り替えます。
    [追加]をクリックすると、新しい空のグループを作成します。
    [削除]をクリックすると、現在選択されているグループが削除されます。
    テキストを変更し[名前を変更]をクリックすると、現在選択されているグループの名前を変更します。
  ・中央左側はフィールド エディタのペインです。
    上側のテキストボックスにフィールド名(フィールドを識別する文字列)を入力し、その下のコンボボックスにフィールド値(フォーマットに実際に埋め込まれる文字列)を入力します。
    [値を追加]をクリックすると、テキストがコンボボックスの要素に追加されます。以降、コンボボックスのドロップダウンから選択すれば値を変更する事ができます。
    [値を削除]をクリックすると、選択中のテキストがコンボボックスの要素から削除されます。
    [+]をクリックすると、新しいフィールドが下に追加されます。
    [▲][▼]をクリックすると、フィールドがそれぞれ上下に移動します。
    右上の×をクリックするとフィールドが削除されますが、フィールド エディタが1つの場合は削除できません。
  ・中央右側はフォーマット エディタのペインです。
    タブ内上側のテキストボックスにフォーマット名を入力し、下側のテキストボックスにフォーマットを入力します。
    フォーマットは ${フィールド名} の形式で記述された部分にフィールド値が埋め込まれます。
    例えば My Name is ${Name}. と記述し、フィールド名 Name に Bill Gates が選択されていると、
    変換後の文字列は My Name is Bill Gates. となります。
    タブを切り替えればフォーマットを切り替える事ができます。
    右上の×をクリックするとフォーマットが削除されますが、フォーマット エディタが1つの場合は削除できません。
    [タブを追加]をクリックすると、空のフォーマットが追加されます。
    変換モード(後述)で[更新]をクリックすると、現在のフィールド値で変換後の文字列を更新します。
  ・左下の[変換(V)]ボタンをクリックすると変換モードになり、フォーマットの文字列にフィールド値が埋め込まれます。
    起動時には編集モードになっています。
      ・変換モードではボタンが[編集(E)]になり、クリックすると編集モードに戻ります。
      ・変換モードではグループの追加／削除／名前の変更、フィールドの追加／移動／削除、フィールド名の変更、フォーマットの追加、フォーマット名／フォーマット内容の変更、新規作成は実行できません。
  ・[コピー(C)]ボタンをクリックすると、フォーマット又は変換後の文字列がクリックボードにコピーされます。
  ・右下の[新規作成(I)]をクリックすると、全てのグループが削除され、空のグループが作成されます。
  ・[読み込み(L)]をクリックすると、保存されたデータがファイルから読み込まれます。
  ・[保存(S)]をクリックすると、作成されたデータがファイルに保存されます。
    ファイルは PFormat.exe と同じ場所に PFormat.xml という名前で保存されます。
・設定
  ・設定ファイルは %LOCALAPPDATA%\Project_Flower\PFormat.exe_Url_*\user.config (* はランダムな文字列) に保存されます。
  ・新規グループの名前は DefaultGroupName の値で記録されます。
  ・ファイル保存先の設定は SaveFileName の値で記録されます。
